## Why Scani

The same TypeScript codebase runs three ways: fully self-hosted, against a hosted
data-provider, or as a managed service. MIT licensed. No telemetry. No vendor lock-in.

<CardGrid stagger>
  <Card title="Self-host in one command" icon="rocket">
    `bun run dev:stack` boots the full stack against Postgres, Redis, MinIO, and Mailpit.
    See the [Quickstart](/quickstart/).
  </Card>
  <Card title="Three tiers, same binaries" icon="setting">
    Pick fully self-hosted, semi-managed, or fully managed by setting two env vars.
    See the [Tier model](/self-hosting/tier-model/).
  </Card>
  <Card title="Provider integrations included" icon="puzzle">
    Binance, Kraken, Coinbase, IBKR, Wise, Etherscan, Helius, CoinGecko, OpenAI, and more.
    See [Provider integrations](/concepts/integrations/).
  </Card>
  <Card title="No telemetry, ever" icon="approve-check">
    The OSS distribution sends no install IDs, no usage counters, no version-check beacons.
    See [Privacy & telemetry](/concepts/privacy/).
  </Card>
