# PWA (Progressive Web App) Setup - October 9, 2025

## Mobile Fixes Implemented

### 1. Viewport Configuration
Fixed horizontal scrolling and zoom-on-input issues with enhanced viewport meta tag:

```html
<meta 
  name="viewport" 
  content="width=device-width, initial-scale=1.0, maximum-scale=1.0, user-scalable=no, viewport-fit=cover" 
/>
```

**Fixes:**
- ✅ Prevents page from being wider than screen
- ✅ Prevents zoom when focusing input fields
- ✅ Prevents horizontal scrolling
- ✅ Supports safe area insets on notched devices (iPhone X+)

### 2. CSS Mobile Optimizations
Enhanced `index.css` with mobile-specific fixes:

**Horizontal Scroll Prevention:**
```css
html, body, #root {
  overflow-x: hidden;
  max-width: 100vw;
}
```

**Input Zoom Prevention:**
```css
input, textarea, select {
  font-size: 16px; /* iOS doesn't zoom if font-size >= 16px */
}
```

**iOS Optimizations:**
```css
body {
  -webkit-overflow-scrolling: touch;
  -webkit-text-size-adjust: 100%;
  overscroll-behavior-y: none; /* Prevents pull-to-refresh */
}
```

## Development

### Testing PWA Locally

1. Build the app:
```bash
cd apps/frontend
bun run build
```

2. Serve the production build:
```bash
bunx serve dist
```

3. Test in mobile device or Chrome DevTools:
   - Chrome DevTools → Application → Service Workers
   - Chrome DevTools → Application → Manifest
   - Chrome DevTools → Lighthouse → Run PWA audit

### Generating Production Icons

The current icons are SVG placeholders. For production:

**Option 1: Online Tool**
1. Go to https://realfavicongenerator.net/
2. Upload your logo/icon
3. Configure settings for all platforms
4. Download and replace icons in `/public/icons/`

**Option 2: ImageMagick**
```bash
# Convert SVG to PNG at different sizes
convert public/icons/icon.svg -resize 192x192 public/icons/icon-192x192.png
convert public/icons/icon.svg -resize 512x512 public/icons/icon-512x512.png
# etc.
```

**Option 3: Design Tool**
- Export from Figma/Sketch at all required sizes
- Ensure icons have proper padding (safe area)

### Service Worker Caching Strategy

**Static Assets (JS, CSS, Images):**
- Strategy: Cache-first
- Loads instantly from cache
- Updates in background

**API Calls:**
- Strategy: Network-first
- Fresh data when online
- Fallback to cache when offline

**HTML Pages:**
- Strategy: Network-first
- Always try to get latest version
- Fallback to cached version if offline

## Troubleshooting

### Service Worker Not Updating
```javascript
// Force update in browser console
navigator.serviceWorker.getRegistrations().then(registrations => {
  registrations.forEach(r => r.unregister());
});
location.reload();
```

### iOS Doesn't Show "Add to Home Screen"
- Must be in Safari (not Chrome/Firefox on iOS)
- Must be HTTPS (http won't work)
- User must manually tap Share → Add to Home Screen

### Manifest Not Loading
- Check file is at `/manifest.json` (root of public folder)
- Verify MIME type is `application/json`
- Check browser console for errors
